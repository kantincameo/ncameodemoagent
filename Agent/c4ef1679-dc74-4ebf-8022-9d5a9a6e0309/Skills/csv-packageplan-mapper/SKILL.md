# CSV PackagePlan Mapper

Parse an uploaded CSV, intelligently map each column to the PackagePlan target structure,
apply validations and defaults, and return an array of fully-formed PackagePlan JSON objects.

## Target PackagePlan Structure

```json
{
  "id": "<auto-generated GUID>",
  "businessId": "string",
  "packageId": "<from CSV id column>",
  "locationId": "string",
  "packageCode": "string",
  "packageName": "string",
  "description": "string",
  "packageCategory": "Default",
  "businessName": "Default",
  "price": 0.0,
  "priceSource": "FromSourceFile",
  "onlineBookingEnabled": true,
  "taxGroup": null,
  "centerTaxId": null,
  "hasPackageSales": true,
  "hasServices": true,
  "hasServiceDiscount": true,
  "hasFreeProducts": false,
  "hasBundledProducts": false,
  "hasForms": false,
  "hasClasses": false,
  "hasWorkshops": false,
  "hasDayPackage": false,
  "benefits": [],
  "ownerType": "Organization",
  "ownerName": "string",
  "isActive": true,
  "createdAt": "<current UTC ISO8601>",
  "updatedAt": "<current UTC ISO8601>",
  "_etag": null,
  "_ts": null
}
```

---

## Step 1 — Parse the CSV

Read the uploaded CSV file and extract:
- All column header names (row 1)
- All data rows

Trim whitespace from all header names and values.

---

## Step 2 — Intelligent Column Mapping

Map each CSV column to a target field using fuzzy/semantic matching. A column maps to a
target field if it is a recognizable variant of that field name — abbreviations, alternate
casing, synonyms, or partial matches all count.

### Mapping Reference Table

| CSV Column Examples                          | Maps To                    |
|----------------------------------------------|----------------------------|
| id, Id, ID, sourceId, source_id              | packageId                  |
| packageName, packName, Name, Title, PkgName, pkg_name | packageName       |
| description, desc, details, about, summary   | description                |
| price, cost, amount, fee, rate, Price        | price                      |
| packageCode, code, pkgCode, pkg_code, Code   | packageCode                |
| packageCategory, category, type, Category    | packageCategory            |
| businessId, bizId, orgId, business_id        | businessId                 |
| businessName, bizName, company, Business     | businessName               |
| locationId, location, locId, loc_id          | locationId                 |
| ownerName, owner, org, organization          | ownerName                  |
| ownerType, orgType, owner_type               | ownerType                  |
| isActive, active, status, enabled            | isActive                   |
| onlineBookingEnabled, onlineBooking, booking | onlineBookingEnabled       |
| serviceName, serviceNameRaw, service         | benefits[].serviceNameRaw  |
| serviceId, svcId, service_id                 | benefits[].serviceId       |
| totalCredits, credits, lessons, sessions, qty | benefits[].totalCredits   |
| hasPackageSales, packageSales                | hasPackageSales            |
| hasServices, services                        | hasServices                |
| hasServiceDiscount, serviceDiscount, discount| hasServiceDiscount         |
| hasFreeProducts, freeProducts                | hasFreeProducts            |
| hasBundledProducts, bundledProducts          | hasBundledProducts         |
| hasForms, forms                              | hasForms                   |
| hasClasses, classes                          | hasClasses                 |
| hasWorkshops, workshops                      | hasWorkshops               |
| hasDayPackage, dayPackage                    | hasDayPackage              |

For boolean fields, accept these values (case-insensitive):
- True: `true`, `1`, `yes`, `y`, `on`
- False: `false`, `0`, `no`, `n`, `off`

For columns that cannot be confidently matched to any target field, skip them and note
them in the unmapped columns list.

---

## Step 3 — Apply Validations and Defaults

For every row, apply these rules in order:

### Identity Fields
- **id**: Always auto-generate a new GUID (never use any CSV value for this field)
- **packageId**: Use the CSV `id` column value. If the column is missing or empty → generate a new GUID

### Pricing
- **price**: Must be a numeric value ≥ 0. Strip currency symbols ($, £, €) before parsing.
  If missing, empty, null, or non-numeric → default `0.0`
- **priceSource**: Always set to `"FromSourceFile"` regardless of CSV content

### String Fields with Defaults
- **packageCategory**: If missing or empty → default `"Default"`
- **businessName**: If missing or empty → default `"Default"`
- **ownerType**: If missing or empty → default `"Organization"`
- **packageName**: If missing or empty → use `"Unknown Package"`
- **description**: If missing → use empty string `""`
- **packageCode**: If missing → use empty string `""`
- **businessId**: If missing → use empty string `""`
- **locationId**: If missing → use empty string `""`
- **ownerName**: If missing → use empty string `""`

### Boolean Fields
- **isActive**: If missing → default `true`
- **onlineBookingEnabled**: If missing → default `true`
- **hasPackageSales**: If missing → default `true`
- **hasServices**: If missing → default `true`
- **hasServiceDiscount**: If missing → default `true`
- **hasFreeProducts**: If missing → default `false`
- **hasBundledProducts**: If missing → default `false`
- **hasForms**: If missing → default `false`
- **hasClasses**: If missing → default `false`
- **hasWorkshops**: If missing → default `false`
- **hasDayPackage**: If missing → default `false`

### Timestamps
- **createdAt**: Always set to current UTC datetime in ISO 8601 format (e.g. `2025-07-13T10:30:00Z`)
- **updatedAt**: Always set to current UTC datetime in ISO 8601 format (same value as createdAt)

### Null Fields
- **taxGroup**: Always `null`
- **centerTaxId**: Always `null`
- **_etag**: Always `null`
- **_ts**: Always `null`

### Benefits Array

**Default: `benefits` is always `[]` (a blank array) unless benefits-related columns are present AND have non-empty values.**

Apply these rules in order:

1. **No benefits columns in CSV** — If none of `serviceNameRaw`, `serviceId`, or `totalCredits` (or their recognized variants) exist as columns in the CSV → set `benefits` to `[]`.

2. **Benefits columns exist but all values are empty/blank for a row** — If the benefits-related columns are present but all their values for that row are empty, null, or whitespace → set `benefits` to `[]` for that row.

3. **At least one benefits value is present for a row** — Build one entry:
```json
{
  "serviceNameRaw": "<from CSV serviceName column, or empty string if missing/blank>",
  "serviceId": "<from CSV serviceId column, or empty string if missing/blank>",
  "totalCredits": <from CSV credits/lessons column as integer, or 0 if missing/non-numeric>
}
```
- If `totalCredits` is missing or non-numeric → default `0`
- If `serviceNameRaw` or `serviceId` are missing or blank but `totalCredits` exists → include the entry with empty strings `""` for the missing fields

**Summary:** When in doubt, default `benefits` to `[]`. Only populate it when there is actual mapped data.
